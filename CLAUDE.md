# Repository Instructions for Codex

- Before any work, read /docs/SRS.md and /PLAN.md in full.
- PLAN.md is the single source of truth for progress. Do not track progress anywhere else.
- After completing each step in PLAN.md, mark it [x] and commit that change alone before starting the next step.
- If /PLAN.md does not exist yet, your first task is to generate it from /docs/SRS.md, not to write code.
- Never modify /docs/SRS.md.
- Work one step at a time. Do not implement future steps early, even if related.
- Ask before: deleting files, changing the database schema, adding dependencies not implied by the SRS.
- After each step, summarize what was completed and stop for review before continuing.
